How Vertical is Assigned (AppsFlyer)
Every install is tagged at the campaign level by AppsFlyer first-touch - not just by the creative. A user from a Surveys campaign is permanently tagged as a Survey user even if they later play games or scan receipts. Vertical = acquisition intent. Cross-vertical activity post-install is tracked separately as a positive HQU signal.
Broad Campaigns - Attribution Rule
When a campaign uses general messaging with no specific vertical targeting, the user is tagged by their first meaningful in-app action post-onboarding: Games first = Gaming, Surveys first = Surveys, receipt scan first = Receipts. No action within 48h = Unattributed - excluded from vertical cohorts, included in blended totals.
MMM vs. AppsFlyer - Two Layers
AppsFlyer (user-level): tags each install to channel + vertical. Powers this dashboard.

MMM (channel-level): measures aggregate channel contribution to outcomes without individual user IDs. Used for budget allocation decisions only - does not change how individual users are tagged. Complementary, not competing.

Section C - 90-Day Gate Scorecard
🎯
What is a Gate? A Gate is a pre-scheduled decision checkpoint in the 90-day launch plan. At each gate, performance is measured against mathematical thresholds agreed before the campaign launched - removing emotion from the decision. Three outcomes are possible: DOUBLE DOWN metrics above target, increase budget; HOLD within acceptable range, continue; PIVOT below threshold, change strategy immediately. Any gate in RED triggers an immediate protocol review - do not wait for the next scheduled gate date. Gate 1 (D7) measures acquisition efficiency. Gate 2 (D30) measures retention. Gate 3 (D45) measures partner value. Gate 4 (D90) is the final go/no-go on UK market expansion.
